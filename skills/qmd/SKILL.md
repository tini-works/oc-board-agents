# Goal

Search memory and workspace files to retrieve relevant past context, decisions, and facts.
Use this instead of reading all memory files manually — QMD finds the right files fast.

# Key Facts

- **Binary:** `~/.bun/bin/qmd`
- **Collections:** `memory` (daily notes + MEMORY.md) · `workspace` (all workspace *.md)
- **Index:** `/home/node/.cache/qmd/index.sqlite`
- **BM25:** works immediately (keyword, no GPU needed)
- **Vector search:** requires `qmd embed` (slow on CPU, no GPU in sandbox)
- **Update index:** `qmd update` after adding new files

# Instructions

## 1. Choose the right search command

| Goal | Command |
|---|---|
| Fast keyword lookup | `qmd search "term" -c memory` |
| Semantic / fuzzy concept | `qmd vsearch "concept" -c memory` |
| Best quality (hybrid + rerank) | `qmd query "question" -c memory` |
| Search everywhere | omit `-c` flag |
| Find specific file | `qmd get qmd://memory/2026-03-06.md` |
| List all indexed files | `qmd ls memory` or `qmd ls workspace` |

## 2. Search memory first, workspace second

```bash
# Step 1: search memory collection (daily notes + MEMORY.md)
~/.bun/bin/qmd search "prev-cli" -c memory

# Step 2: if not found, search workspace (skills, plans, configs)
~/.bun/bin/qmd search "prev-cli" -c workspace

# Step 3: if still not found, check MEMORY.md directly
```

## 3. After every session — update index

```bash
~/.bun/bin/qmd update
```

## 4. After major workspace changes — re-embed

```bash
~/.bun/bin/qmd embed
```

## 5. JSON output for structured retrieval

```bash
~/.bun/bin/qmd search "X" --json -n 5
~/.bun/bin/qmd search "X" --all --files --min-score 0.4
```

→ Returns: `memory/2026-03-06.md` with fact "Bun v1.3.10 installed at ~/.bun/bin/bun"

```bash

```

→ Returns: skill files + MEMORY.md entries about c3

## Example 3: Retrieve a specific day's notes

```bash
~/.bun/bin/qmd get qmd://memory/2026-03-06.md
```

# Constraints

- BM25 search is **case-insensitive keyword** — use quotes for exact phrases
- Vector/semantic search (`vsearch`, `query`) requires embeddings — run `qmd embed` first
- GPU not available in sandbox — embedding is slow (CPU only); BM25 works without it
- Always run `qmd update` when memory files change between searches
- Index path: `/home/node/.cache/qmd/` — do not delete
